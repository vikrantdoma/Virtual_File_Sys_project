Virtual File System using Software design patterns in Linux					
This is part 3 of a 5 part project. The following java program to simulates reading from a file containing linux file handling commands (script2.txt). The program parses the text file commands, this project was done implementing the waterfall Software development lifecycle, ALL SOLID programming principles have been applied along with a few software design patterns:- singleton, composite, builder, visitor, adapter, and observer.

This program has been written with the help of eclipse IDE (2019 version).

To run this program:
Simply import the project into the eclipse editor, else you can import each java file manually. 
The path for script2.txt can be changed in the "FileSystemCreator.java" file
